Lesson 2 Notes:


Document Object Model:

The Document Object Model (DOM), is the "tree like structure" of the page.It shows where one command branchs out into more specific and detailed commands for the computer to follow. It turns the tags on a page into elements in a "tree".


HTML and CSS in DOM:

HTML and CSS are both types of languages used in coding. HTML controls the structure of the page. CSS controls the style of the page.


Boxes:

Everything on a webpage or anything that has to do with coding is a box! It doesn't matter if a bubble on a webpage looks like a circle it started out as a box, and then coding made it specified to look like a circle. On a webpage there will be boxes, inside boxes, inside boxes.This has to do with images, borders, buttons, and links to other pages. Even if it doesn't look like a box don't let it fool you, it is.


Lesson 3 Notes:


CSS:
  • CSS stands for "Cascading Style Sheets"

Until code is written and specified to a certain section, that section will look the same and have the same code as the other sections, until told otherwise.

The entire tag of a CSS is called a "declaration". The code written in front of the colon (:) is called the "Property" and the code written behind the colon (:) but infront of the semicolon (;) is called "Value".


The Box Model:

Made up of:(Going from outside of box -> inside the box)
  1. Margins - They are transparent
  2. Border - The border lines around a box
  3. Padding - The background color and things that go around the content
  4. Content- The color of the text, the text itself, and anything that has to do with the content of the box
